UC1: Execute Pipeline Processing

Acesta este fluxul principal prin care imaginile sunt procesate utilizând arhitectura hibridă (Local + Cloud).

Actori: Photographer (Principal), TensorFlow Lite, Flask Server.

Precondiții: Utilizatorul a creat un workflow valid și a selectat cel puțin o imagine din galerie.

Declanșator: Utilizatorul apasă butonul din interfață sau se împlinește condiția dintr-un trigger (ex: finalizarea unui upload în Google Drive).

Flux Principal (Scenariul de succes):

  1. Sistemul inițializează procesarea pe un fir de execuție secundar.
  2. Sistemul invocă TensorFlow Lite pentru a extrage vectorul de trăsături al imaginii (On-Device).
  3. Aplicația trimite vectorul numeric către Flask Server prin API.
  4. Serverul analizează vectorul și returnează parametrii de editare optimi (brightness, contrast, saturation).
  5. Sistemul aplică parametrii primiți asupra imaginii originale.
  6. Sistemul afișează imaginea editată utilizatorului.

Flux Alternativ (Eroare Conexiune):

La pasul 3, dacă serverul nu răspunde în 5 secunde, sistemul aplică un set de filtre implicite (fallback) și notifică utilizatorul că procesarea AI nu este disponibilă.

Postcondiții: Imaginea procesată este disponibilă în memoria temporară pentru revizuire sau export.

UC2: Create Node Workflow

Permite fotografului să definească logica personalizată de editare folosind interfața vizuală.

Actori: Photographer.

Precondiții: Utilizatorul navighează în activitatea NodeEditor.

Declanșator: Utilizatorul accesează meniul de adăugare noduri.

Flux Principal:

  1. Utilizatorul selectează un tip de nod din meniu (ex: Source, StyleTransfer, Output).
  2. Sistemul plasează nodul pe suprafața de lucru (Canvas).
  3. Utilizatorul trage o conexiune (drag-and-drop) între portul de ieșire al unui nod și portul de intrare al altuia.
  4. Sistemul validează compatibilitatea conexiunii.
  5. Utilizatorul repetă pașii până când graful este complet (de la Source la Output).

Flux Alternativ (Validare Ciclu):

La pasul 3, dacă utilizatorul încearcă să creeze o buclă infinită (ex: Nod A -> Nod B -> Nod A), sistemul blochează conexiunea și afișează eroarea "Cycle Detected".

Postcondiții: Un pipeline valid este stocat în memorie, gata de execuție sau salvare.
